vc::Tensor<T> — The Foundation

A GPU-ready, auto-differentiable multidimensional array.

Overview

If you've ever used NumPy or PyTorch, the Tensor class will feel right at home. It is the absolute core of Vecore. You can think of a Tensor as a multi-dimensional array on steroids: not only does it store your raw data (like images, audio, or text embeddings), but it also remembers how it was created.

Every time you add, multiply, or pass a Tensor through a layer, Vecore quietly builds a computational graph in the background. When it's time to learn, a single call to .backward() traverses this graph backwards, calculating the exact gradients needed to update your weights.

Beyond autograd, the Tensor seamlessly bridges the gap between your CPU and GPU. With a simple .to("cuda"), your data is transferred to the GPU, and subsequent operations automatically route through heavily optimized cuBLAS Tensor Cores.

Constructors

Tensor()

Default constructor. Creates an empty tensor with no data and a 0-dimensional shape.

auto t = vc::Tensor<float>();
Tensor(vc::vector<size_t> shape)

Creates a zero-initialized tensor of the given shape on the CPU.

// Create a 2x3 tensor initialized to zeros
auto t = vc::Tensor<float>({2, 3});
static Tensor<T> empty_gpu(const vc::vector<size_t>& shape)

Factory method for creating an uninitialized GPU tensor directly, avoiding CPU allocation.

auto t = vc::Tensor<float>::empty_gpu({128, 128});

Core Methods

Tensor<T> to(const std::string& device) const

Transfers the tensor between "cpu" and "cuda". If the tensor is already on the specified device, it returns a view of itself.

ParameterTypeDescription
deviceconst std::string&The target device, either "cpu" or "cuda".

Returns: Tensor<T> - A tensor on the specified device.

auto cpu_tensor = vc::Tensor<float>({4, 4});
auto gpu_tensor = cpu_tensor.to("cuda");
Tensor<T> operator+(const Tensor<T>& other) const

Element-wise addition with broadcasting support. Creates an AddNode in the autograd computation graph.

ParameterTypeDescription
otherconst Tensor<T>&The tensor to add.

Returns: Tensor<T> - Result of element-wise addition.

auto a = vc::Tensor<float>({2, 2});
auto b = vc::Tensor<float>({1, 2});
auto c = a + b; // b is broadcasted to {2, 2}
Tensor<T> operator-(const Tensor<T>& other) const

Element-wise subtraction with broadcasting. Creates a SubNode in the autograd graph.

ParameterTypeDescription
otherconst Tensor<T>&The tensor to subtract.

Returns: Tensor<T> - Result of element-wise subtraction.

auto c = a - b;
Tensor<T> operator*(const Tensor<T>& other) const

Performs matrix multiplication. Uses cuBLAS on GPU and OpenBLAS on CPU. Creates a MulNode in the autograd graph.

Tip: This is matrix multiplication, not element-wise multiply. Vecore follows the same convention as PyTorch's @ operator.
ParameterTypeDescription
otherconst Tensor<T>&The tensor to multiply with.

Returns: Tensor<T> - The matrix multiplication product.

auto A = vc::Tensor<float>({2, 3});
auto B = vc::Tensor<float>({3, 4});
auto C = A * B; // Shape {2, 4}
Tensor<T> transpose() const

Returns a view with swapped last two dimensions. This is a zero-copy operation that modifies strides.

Returns: Tensor<T> - Transposed view of the tensor.

auto t = vc::Tensor<float>({2, 3});
auto tt = t.transpose(); // Shape {3, 2}
Tensor<T> relu() const

Applies element-wise Rectified Linear Unit (ReLU) activation: max(0, x).

Returns: Tensor<T> - Activated tensor.

auto activated = t.relu();
Tensor<T> sum(const vc::vector<int>& dims, bool keepdim = false) const

Performs sum reduction along specified dimensions.

ParameterTypeDescription
dimsconst vc::vector<int>&Dimensions to reduce.
keepdimboolWhether to retain reduced dimensions with size 1.

Returns: Tensor<T> - Reduced tensor.

auto t = vc::Tensor<float>({2, 3});
auto s = t.sum({1}, false); // Shape {2}
Tensor<T> reshape(const vc::vector<size_t>& new_shape) const

Returns a tensor with a different shape sharing the same underlying data buffer. Number of elements must match.

ParameterTypeDescription
new_shapeconst vc::vector<size_t>&The desired shape.

Returns: Tensor<T> - Reshaped tensor view.

auto t = vc::Tensor<float>({2, 3});
auto flat = t.reshape({6});
size_t numel() const

Returns the total number of elements in the tensor. This is a cached O(1) operation.

Returns: size_t - Total element count.

auto t = vc::Tensor<float>({2, 3, 4});
size_t n = t.numel(); // Returns 24
void backward()

Triggers reverse-mode automatic differentiation. Performs a topological sort of the computation graph from this tensor to all inputs, propagating gradients. Typically called on the scalar loss tensor.

For each node in the graph, it computes local gradients and accumulates them in the parents' ctx->grad.

loss.backward();
void print() const

Prints the tensor's shape, device, and elements to standard output. Useful for debugging.

t.print();

Autograd System

Vecore features a dynamic, reverse-mode automatic differentiation engine. Each differentiable operation dynamically creates a node in the computation graph.

  • Graph Building: Nodes such as AddNode, SubNode, MulNode, ReLU, and ToNode record the history of operations.
  • Backpropagation: Calling backward() initiates a topological sort of the graph, ensuring every node's gradient is computed exactly once.
  • Gradient Accumulation: Computed gradients are accumulated into the respective tensor's ctx->grad buffer.
x (input) ──→ MatMul(x, W) ──→ Add(+bias) ──→ ReLU ──→ output
                  ↑                  ↑
             W (weight)             bias

GPU Memory Management

Vecore avoids the overhead of frequent cudaMalloc and cudaFree calls by utilizing a custom CachingAllocator.

  • Memory Pooling: Freed GPU memory is cached and pooled for immediate reuse.
  • Performance: Eliminates synchronization stalls associated with raw CUDA allocations.
  • RAII Semantics: Managed automatically via the GPUData RAII wrapper, which returns memory to the allocator upon destruction.
