What We Built From Scratch The Stack

Every component — from raw heap allocation to GPU kernel dispatch — was written in C++ with zero third-party ML libraries.

🗂️

mstd::vector<T>

A custom dynamic array built from scratch with raw heap pointers, amortized O(1) push_back via doubling, and the Rule of Five for safe copy/move semantics.

C++ Memory
📐

Tensor<T>

N-dimensional array with shape, stride-based indexing, and zero-copy transpose. Supports both CPU (RAM) and GPU (VRAM) memory backends simultaneously.

C++ / CUDA
🔗

Autograd Engine

Dynamic computation graph built during the forward pass. AutogradNode polymorphism + Depth-First topological sort for correct reverse-mode differentiation.

Calculus
➕

AddNode / MulNode

Differentiable add and matrix multiply operations. MulNode backward computes dL/dA = dL/dC · Bᵀ and dL/dB = Aᵀ · dL/dC via gradient accumulation.

Backprop
⚡

ReLU Activation

ReLUNode stores the mask of positive pre-activations during the forward pass and gates gradients through it during the backward pass.

Non-linearity
🧱

nn::Dense Layer

Fully-connected layer with Xavier weight initialization. operator() runs (x · W) + b and wires the result into the autograd graph automatically.

Neural Layer
📉

MSELoss & CrossEntropyLoss

Two pluggable loss functions. CrossEntropy runs Softmax + Negative Log-Likelihood. Its backward gradient is elegantly just P_i − Target_i.

Loss Functions
🏃

optim::SGD

Stochastic Gradient Descent with zero_grad() to clear accumulators and step() to apply weight = weight − lr · grad. GPU weights are re-synced after each update.

Optimizer
🖼️

MNIST PNG Loader

stb_image reads raw .png files from disk. Pixels are normalized to [0,1] and labels are one-hot encoded into [1×10] target tensors.

Data Pipeline
🔢

OpenBLAS Integration

cblas_sgemm dispatches AVX-optimized matrix multiplication on the CPU. Handles transposed views correctly via stride inspection.

CPU Math
🎮

GPUData<T> + .to("cuda")

RAII wrapper around cudaMalloc/cudaFree. Tensor::to() uses cudaMemcpy to transfer data across the PCIe bus between RAM and the RTX 5050's VRAM.

CUDA
🚀

cuBLAS Matrix Multiply

cublasSgemm dispatches matrix multiplication directly to the GPU's CUDA cores. Column-major/row-major mismatch handled by swapping A↔B in the API call.

GPU Acceleration
Built entirely in C++ · cuBLAS + OpenBLAS · NVIDIA RTX 5050 · MNIST Dataset · No ML libraries used
